# Markdown 报告一键转精美 PPT+PDF，我写了一个开源技能包

> 一份 Markdown 写的研究报告，三分钟出三种格式（PPTX + PDF + DOCX），配色统一、排版精美、可跨平台使用。

---

## 一、痛点：为什么现有的方案都不够好？

做研究、写报告的时候，Markdown 是最高效的写作工具。但交付的时候，甲方/老板/客户要的是 **PPT 和 PDF**。

市面上的方案我基本都试过：

| 方案 | 问题 |
|------|------|
| **Pandoc 裸转** | 纯白幻灯片，无配色无设计，排版「工业化」 |
| **商汤 PPT Skills** | 依赖外部 API，图生成失败率极高，稳定性堪忧 |
| **Playwright / Puppeteer 打 PDF** | Windows 环境坑多，playwright 安装频繁失败 |
| **Canva / 在线工具** | 手动排版耗时，且不能与 AI 工作流衔接 |

核心矛盾：**写得快 ≠ 交付快**。Markdown 写完了，排版还要半天。

---

## 二、解决方案：`md-to-ppt-pdf`

一个 **纯 Python、零外部服务依赖、离线可用** 的 Markdown 转换工具链。设计理念借鉴了 GitHub 上 5700+ Star 的开源项目 **html-ppt-skill**（也是 Marvis PPT 能力的技术基础）。

### 三条命令出成品

```bash
# PPTX — 12页精美幻灯片，Navy+Gold 企业级配色
python scripts/generate_ppt.py

# HTML — Markdown 转美化网页，CSS 增强注入
pandoc report.md -o report.html --standalone
python scripts/inject_css.py report.html

# PDF — Edge/Chrome 无头模式，A4 精排版
msedge --headless --disable-gpu --print-to-pdf=report.pdf report.html
```

输出效果：

```
report.md                      ← 你的 Markdown 源文件
    │
    ├──→ [python-pptx 引擎] ──→ report.pptx   封面/目录/卡片/漏斗/时间线
    │
    └──→ [MD→HTML→CSS增强] ──→ report.pdf    斑马纹表格/金边引用/配色统一
```

## 三、核心技术：Token 驱动设计系统

不同于硬编码样式，这个工具采用 **DesignTokens 设计令牌** 机制——一套变量控制全局配色、字体、间距。改一个变量，整套 PPT 和 PDF 同步换皮。

```python
class DesignTokens:
    COLOR_PRIMARY    = RGBColor(0x0D, 0x21, 0x37)   # Navy Blue  主色
    COLOR_SECONDARY  = RGBColor(0xD4, 0xAF, 0x37)   # Gold       强调色
    COLOR_ACCENT     = RGBColor(0x2E, 0x86, 0xAB)   # Teal       数据色
    COLOR_BG_LIGHT   = RGBColor(0xF5, 0xF3, 0xEE)   # 暖灰       背景
    COLOR_CARD_BG    = RGBColor(0xFF, 0xFF, 0xFF)   # 白色       卡片
```

内置 **11 种布局模板**，覆盖研究报告的典型场景：

| 布局模板 | 用途 |
|----------|------|
| `slide_cover` | 封面（标题 + 副标题 + 日期） |
| `slide_toc` | 目录页 |
| `slide_3_conclusions` | 三栏核心结论卡片 |
| `slide_market_size` | 数据表格 + 指标高亮 |
| `slide_competitive` | 双栏竞品对比 |
| `slide_monetization` | 三层变现漏斗图 |
| `slide_differentiation` | 四卡差异化分析 |
| `slide_90day_plan` | 时间线里程碑 |
| `slide_risk` | 风险预案表 |
| `slide_thanks` | 致谢尾页 |

## 四、PDF 也不含糊

PDF 不是简单的「打印网页」，而是经过 CSS 增强注入的专业输出：

- **表格斑马纹** — 偶数行浅灰底，表头 Navy Blue 底色白字
- **标题层级** — h1 金色下划线，h2 金色左竖线，h3 青色
- **引用块** — 金色左边框 + 暖灰背景
- **A4 打印优化** — `@page { size: A4; margin: 20mm }`
- **打印时自动去背景** — 省墨，白底黑字

## 五、不只是 WorkBuddy 专用——跨智能体可迁移

这个技能包的核心是两个自包含的 Python 脚本。不只适用于 WorkBuddy，也能接入其他 AI 编程助手：

| 平台 | 适配方式 |
|------|----------|
| **Cursor** | 复制指令到 `.cursorrules` |
| **Claude Code** | 改写为 `.claude/commands/md-to-ppt.md` slash command |
| **GitHub Copilot** | 写入 `.github/copilot-instructions.md` |
| **Cline / Roo Code** | 添加 `.clinerules` 自定义指令块 |
| **任意 IDE** | 直接 `python scripts/generate_ppt.py` 跑脚本 |

只要环境里有 `python-pptx`，任何 AI 工具都能调用。

## 七、快速上手

### 环境要求

```bash
pip install python-pptx          # 必需
# 以下可选
pandoc                           # MD→HTML（或用任意 Markdown 渲染器替代）
```

### 三步出成品

```bash
# 1. PPTX
python scripts/generate_ppt.py

# 2. HTML + CSS 增强
pandoc report.md -o report.html --standalone
python scripts/inject_css.py report.html

# 3. PDF（Windows）
msedge --headless --disable-gpu --print-to-pdf=report.pdf report.html

# 或一键：
make all MD="你的报告.md"
```
